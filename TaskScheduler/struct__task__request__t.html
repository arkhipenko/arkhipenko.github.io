Detailed Description

Structure for thread-safe task operation requests.

This structure is used to call Scheduler::requestAction() and provide parameters to the respective API method in a thread-safe manner. When _TASK_THREAD_SAFE is enabled, this structure allows tasks to be controlled safely from interrupt contexts or other threads by queuing requests that are processed atomically during the scheduler's execute() cycle.

The structure contains the request type and up to 5 parameters that correspond to the parameters of the target Task or StatusRequest method being called. The scheduler will unpack these parameters and call the appropriate method during its execution.

Usage example:

_task_request_t request;
request.req_type = TASK_REQUEST_ENABLE;
request.object_ptr = &myTask;
// param1-param5 not needed for enable()
scheduler.requestAction(&request);
